# 后端错误输出到控制台-测试概述

## 测试目标

验证 `scripts/start-dev.sh` 在后台启动后端时，会把后端 stderr 同步打印到当前控制台，并且仍然保留到后端日志文件。

## 验证命令

```bash
bash scripts/test_start_dev_backend_console_logging.sh
```

## 结果

通过

输出：

```text
PASS: 后端 stderr 已同步打印到控制台并写入日志
```

## 覆盖点

- 假后端进程向 stderr 输出 `模拟后端报错`
- 当前控制台 stderr 能收到该输出
- `/tmp/kanban-backend-<port>.log` 仍保留该输出

## 说明

本次只调整后端 stderr 的控制台镜像，不改变前端日志行为，也不把服务切成前台常驻。
