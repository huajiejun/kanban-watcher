# 手机模式快照补拉修复测试概述

## 本次目标

- 复现并修复手机模式弹窗在 `workspace_snapshot` 更新后仍停留旧消息的问题
- 将“消息版本变化触发补拉”的判定收敛到共享 realtime helper
- 确认 PC 端接入同一补偿策略后不引入 session websocket 重连回归

## 执行测试

### 1. 共享 helper 单测

命令：

```bash
npm test -- --run tests/realtime-sync.test.ts
```

结果：

- 通过，`1` 个测试文件，`8` 条用例全部通过

### 2. 手机模式回归测试

命令：

```bash
npm test -- --run tests/todo-integration.test.ts -t "refreshes the mobile card dialog when workspace snapshot updates the selected workspace timestamp|uses realtime.base_url for the mobile card|requests filtered latest messages for the mobile card dialog|switches the mobile card session websocket when workspace snapshot updates latest_session_id"
```

结果：

- 通过，命中的 `5` 条用例全部通过，其他用例被跳过
- 新增用例验证：弹窗已打开且 session 未变化时，只要 `workspace_snapshot` 更新了选中工作区的消息版本，就会补拉一次 `latest-messages`

### 3. PC 端回归测试

命令：

```bash
npm test -- --run tests/workspace-home.test.ts -t "refreshes the active workspace messages when workspace snapshot updates its message version|does not reconnect the active session websocket when workspace snapshot leaves the active session unchanged|delegates mobile realtime startup to the embedded card"
```

结果：

- 通过，命中的 `3` 条用例全部通过
- 运行中有两条 `markWorkspaceSeen` 的测试 stderr 提示，因为测试桩未覆盖 `/seen` 接口，但不影响断言结果

## 结论

- 手机模式在 `workspace_snapshot` 更新消息版本时会主动补拉当前弹窗消息，不再只依赖 session websocket 增量
- PC 端复用同一判定逻辑，并保持“session 未变时不重连 websocket”的原有行为
- 本次验证范围内未发现新的回归
