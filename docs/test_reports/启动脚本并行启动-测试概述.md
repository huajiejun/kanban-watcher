# 启动脚本并行启动-测试概述

## 测试目标

验证 `scripts/start-dev.sh` 满足以下行为：

- 前后端都以后台进程启动
- 启动脚本不会挂在前端或后端上
- 后端 stderr 会持续打印到当前控制台
- 前后端 PID 文件记录的是实际服务进程

## 验证命令

```bash
bash scripts/test_start_dev_backend_console_logging.sh
```

## 结果

通过

输出：

```text
PASS: 前后端已后台启动，脚本快速返回，后端 stderr 已同步打印到控制台并写入日志
```

## 覆盖点

- 假后端进程会把自身 PID 写入临时文件，并向 stderr 输出 `模拟后端报错`
- 假前端进程会把自身 PID 写入临时文件
- `/tmp/kanban-dev/backend-<port>.pid` 与假后端 PID 一致
- `/tmp/kanban-dev/frontend-<port>.pid` 与假前端 PID 一致
- 启动脚本在 5 秒内返回，说明没有卡在前端或后端前台进程上
- `/tmp/kanban-backend-<port>.log` 保留了后端 stderr
