# 预制按钮上下文历史放宽-测试概述

## 测试目标

验证快捷按钮 LLM 上下文从“按长度决定是否拼接历史”改为“直接使用最后 3 条消息，并标记顺序”。

## 执行记录

### 1. 核心上下文测试

执行命令：

```bash
npm test -- tests/quick-buttons-llm.test.ts tests/quick-buttons.test.ts
```

结果：

- 2 个测试文件全部通过
- 37 个测试全部通过

验证点：

- 不再依赖最后一条消息的长度阈值
- 上下文直接取最后 3 条消息
- 上下文中包含顺序标记，如 `第1条`、`第2条`、`第3条`
- 上下文同时保留用户与 AI 消息角色信息

### 2. 调用侧回归

执行命令：

```bash
npm test -- tests/workspace-home.test.ts
```

结果：

- 与本次改动直接相关的调用链未出现新的类型或运行时报错
- 该测试文件仍存在 1 个既有失败：`uses the full desktop width without centering the workspace shell`

## 结论

本次“上下文历史放宽”改动已经通过核心定向测试验证。`workspace-home` 仍有 1 个历史失败断言，与本次上下文逻辑改动无直接关联。
