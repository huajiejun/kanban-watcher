# 网页版工作区-测试概述

## 测试日期

2026-03-25

## 测试目标

验证网页版工作区主入口的关键链路：

- 首页 `/` 作为默认主入口可正常打开
- 桌面端为左侧状态栏 + 右侧工作区窗格布局
- 手机端退回 Home Assistant 卡片交互
- 运行中窗格支持队列状态、取消队列和同步刷新
- `/preview` 路由继续保留卡片预览页

## 执行命令

```bash
npm test -- tests/workspace-home.test.ts tests/workspace-page-state.test.ts tests/workspace-conversation-pane.test.ts tests/workspace-section-list.test.ts tests/playground.test.ts
npm run build

```

## 测试结果

- `tests/workspace-home.test.ts` 通过，覆盖桌面端同步刷新、手动关闭抑制、发送消息、运行态队列、取消队列
- `tests/workspace-page-state.test.ts` 通过
- `tests/workspace-conversation-pane.test.ts` 通过
- `tests/workspace-section-list.test.ts` 通过
- `tests/playground.test.ts` 通过
- `npm run build` 通过
- 本地浏览器验证通过，已生成桌面端、手机端和 `/preview` 路由截图

## 关键验证点

- 首页默认路由进入网页工作区，而不是旧预览页
- 桌面端点击左侧工作区卡片后，右侧会打开对应会话窗格
- 运行中窗格会带上队列状态，并支持“取消队列”
- 手机端首页退回卡片模式，不展示桌面端多窗格布局
- `/preview` 路由仍然能展示卡片预览页
