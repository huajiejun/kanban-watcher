# 开发服务器子路径访问-测试概述

## 测试目标

验证前端开发服务器在 `https://dev.huajiejun.cn/<前端端口>/` 子路径下访问时，能够正确生成静态资源路径、HMR 连接路径，以及预览页/工作区页的路由与 API 基础路径。

## 测试命令

```bash
npm test -- tests/web-subpath-config.test.ts
npm test -- tests/web-entry.test.ts
bash scripts/test_start_dev_subpath_env.sh
```

## 测试结果

1. `tests/web-subpath-config.test.ts` 通过
   - 验证 `createSharedViteConfig()` 会为 `/<端口>/` 子路径生成 `base`
   - 验证 HMR host/protocol/clientPort/path 可由环境变量注入
   - 验证预览页会从 `https://dev.huajiejun.cn/<端口>/` 自动推导 `baseUrl`
2. `tests/web-entry.test.ts` 通过
   - 验证 `/16473/preview` 会识别为预览页
   - 验证 `/16473/` 会识别为工作区页
3. `scripts/test_start_dev_subpath_env.sh` 通过
   - 验证 `start-dev.sh` 会给前端注入 `VITE_DEV_BASE_PATH`
   - 验证 `start-dev.sh` 会给前端注入 `VITE_BASE_URL`
   - 验证 `start-dev.sh` 会给前端注入 HMR 子路径环境变量

## 结论

当前 `start-dev.sh` + Vite 配置已经支持通过 `https://dev.huajiejun.cn/<前端端口>/` 访问对应 worktree 的前端开发服务，并与现有 Nginx 动态端口代理规则保持一致。
