# 统一消息排序规则-测试概述(中文)

## 测试目标

验证手机端与电脑端统一使用同一套消息排序规则后，核心排序优先级符合预期：

- 同 process 按 `entryIndex`
- 不同 process 优先按 `messageId`
- 再按 `timestamp`
- 最后按 identity 稳定兜底

## 执行命令

### 1. 共享排序规则测试

```bash
npm test -- --run tests/dialog-messages.test.ts
```

结果：通过

### 2. 手机端相关回归

```bash
npm test -- --run tests/todo-integration.test.ts -t "mobile card|workspace snapshot|latest messages"
```

结果：通过

### 3. 桌面端全量回归

```bash
npm test -- --run tests/workspace-home.test.ts
```

结果：失败（5 条既有失败）

失败集中在：

- dev server preview 控件
- file browser path 断言
- `markWorkspaceSeen` 相关 fetch mock

这些失败与本次消息排序 comparator 调整不直接相关，本次改动没有新增新的排序相关失败。
