## 额外观察

补跑以下命令时发现 `tests/kanban-watcher-card.test.ts` 存在一批与本次修复无关的历史失败：

```bash
npm test -- tests/kanban-watcher-card.test.ts
```

现象：

- 多个旧用例直接从 `card.shadowRoot` 查询弹窗内部节点
- 这些节点实际位于 `workspace-conversation-pane` 的 shadow DOM 内
- 因此出现 `.message-input`、`.message-tool-button`、`.dialog-feedback` 等查询为空的失败

该问题已单独记录到 `docs/issues/测试-移动端卡片全量测试穿透ShadowRoot失效.md`，未在本次修复中扩散处理。
