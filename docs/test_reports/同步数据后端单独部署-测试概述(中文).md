# 同步数据后端单独部署-测试概述

## 测试目标

- 验证运行角色默认值与主/业务后端分流规则生效
- 验证 worker 模式下不会暴露 realtime 能力
- 验证 `/api/info` 会返回本地 realtime 信息且不破坏既有结构
- 验证前端板级 websocket 与会话 websocket 都会统一连接主后端
- 验证命令入口改造未破坏既有 `cmd/kanban-watcher` 基线

## 已执行命令

1. `go test ./internal/config -run 'TestApplyDefaults|TestRuntimeRole' -v`
   - 结果：PASS
   - 覆盖：运行角色默认值、显式角色保留、默认 realtime 地址

2. `go test ./internal/server -run 'TestHandleInfoProxiesVibeInfo|TestHandleRealtimeUnavailableReturnsServiceUnavailable' -v`
   - 结果：PASS
   - 覆盖：`/api/info` 扩展 realtime 字段、worker realtime 不可用返回 `503`

3. `go test ./cmd/kanban-watcher -v`
   - 结果：PASS
   - 覆盖：命令路由、`sync-now`、`headless`、轮询结果处理等既有入口基线

4. `npm test -- --run tests/workspace-home.test.ts -t 'uses board websocket snapshots to update the workspace list in api mode'`
   - 结果：PASS
   - 覆盖：板级 websocket 改为连接 `ws://127.0.0.1:7778/api/realtime/ws`

5. `npm test -- --run tests/workspace-home.test.ts -t 'uses websocket for the active pane and appends realtime session messages'`
   - 结果：PASS
   - 覆盖：会话 websocket 改为连接主后端 realtime 地址，包含 `session_id`

## 关键验证结论

- `http_api.port == 7778` 时默认角色为 `main`
- 非 `7778` 端口默认角色为 `worker`
- `worker` 默认 realtime 地址仍指向 `http://127.0.0.1:7778`
- `worker` 下 `/api/realtime/ws` 明确返回 `503 realtime disabled for worker role`
- `/api/info` 仍保留 `data.config.preview_proxy_port`，同时补充本地 `runtime` 与 `realtime` 信息
- 前端板级与会话级 websocket 已统一改连主后端 realtime 地址

## 额外观察

执行整份 `tests/workspace-home.test.ts` 时，存在 4 个与本次改动无直接关联的既有失败：

- `stores the returned dev-server execution process and uses detail status to show running controls`
- `shows dev server controls in the active pane header and opens a preview drawer with iframe`
- `resolves the file browser path from the api before opening the desktop folder preview`
- `updates the pane header back to idle after the dev-server stop request succeeds`

这些失败主要集中在 dev server 预览按钮与文件按钮断言，不阻塞本次“主后端统一 realtime / worker 仅 HTTP”能力验证。
