# 工作区共享布局实时同步-测试概述

日期：2026-03-25

## 本次范围

- 服务级共享布局 `GET/PUT /api/workspace-view`
- `workspace_view_updated` websocket 广播
- 前端启动阶段远端布局 hydrate
- 前端收到共享布局实时事件后的页面收敛
- `dismissedAttentionIds` 本地持久化

## 执行命令

```bash
go test ./...
npm test -- tests/workspace-home.test.ts tests/workspace-page-state.test.ts
```

## 结果

- `go test ./...` 通过
- `tests/workspace-home.test.ts` 通过
- `tests/workspace-page-state.test.ts` 通过

## 关键验证点

- 后端新增共享布局读写接口后，命令行全量 Go 测试通过，说明路由注册和 realtime 事件扩展没有破坏既有服务
- 前端启动时会请求 `/api/workspace-view`，并用远端共享布局覆盖本地旧布局
- 前端收到 `workspace_view_updated` 事件后，会同步 `openWorkspaceIds`、`activeWorkspaceId` 和 `dismissedAttentionIds`
- `dismissedAttentionIds` 会写入本地 `localStorage`
- 现有 attention 自动弹出、running 不误弹、桌面多 pane 等既有测试继续通过

## 剩余风险

- 当前没有补浏览器端真实双窗口联调截图，跨端实时同步主要由单测和后端 websocket 测试覆盖
- 目前采用 `last-write-wins`，多端同时高频编辑布局时仍以后端最后一次成功写入为准
