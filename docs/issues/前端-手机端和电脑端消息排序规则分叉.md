# 前端-手机端和电脑端消息排序规则分叉

## 现象

- 同一批 realtime 消息在手机端和电脑端展示顺序不一致。
- 典型表现是“后发的消息排在前面”。

## 根因

- 电脑端 `workspace-home` 使用一套本地 comparator：
  - 同 process 优先按 `entryIndex`
  - 再按 `messageId`
  - 最后按 `timestamp`
- 手机端 `kanban-watcher-card` 使用另一套本地 comparator：
  - 主要按 `timestamp`
  - 再按 identity 兜底

两端各自对 realtime 追加消息做重排，因此排序差异会直接体现在 UI。

## 处理方向

- 把消息排序规则下沉到共享消息工具层。
- 手机端和电脑端统一调用同一个 comparator。
