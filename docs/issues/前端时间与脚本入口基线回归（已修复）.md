# 前端时间与脚本入口基线回归（已修复）

## 发现日期

- 2026-03-24

## 问题概述

在停止功能联调通过后执行基础回归，发现有两类基线问题：

1. 卡片相对时间测试回归  
   部分工作区缺少 `workspace.updated_at` 时，卡片统一回落为 `recently`，已完成工作区也没有继续使用旧字段 `completed_at`。

2. `go test ./...` 无法完整执行  
   `scripts` 目录下存在两个 `package main` 文件，位于同一包路径，导致全量测试编译阶段直接失败。

## 根因分析

### 1. 时间显示回归

- 前端卡片时间源已改为优先使用 `latest_process_completed_at`
- 但预览/Hass 数据仍有一部分只提供 `completed_at`
- 缺少工作区级时间字段时，也没有继续回退到实体级 `updated_at`

### 2. 全量测试入口回归

- `scripts/check_kw_msg_contexts.go`
- `scripts/run_kw_sync_once.go`

两个脚本都放在 `scripts` 包根目录，`go test ./...` 会把它们当成同一个包进行编译，产生重复 `main` 定义。

## 处理结果

### 时间显示

- 在卡片时间计算中补回 `completed_at` 兼容
- 在工作区级时间为空时，增加实体级 `updated_at` 兜底

### 脚本入口

- 将两个脚本拆分到独立目录：
  - `scripts/check_kw_msg_contexts/main.go`
  - `scripts/run_kw_sync_once/main.go`

## 验证结果

```bash
go test ./...
npm test
npm run build
```

结果：

- 全部通过
