# 本地弹框通知设计

## 1. 概述

**目标**：在 kanban-watcher 电脑端实现分级超时弹框通知，提醒用户及时处理需要关注的工作区。

**核心需求**：
- 超时后弹框，用户点击"查看"打开浏览器，点击"忽略"关闭
- 分级阈值：待审批 5 分钟，未读消息 10 分钟
- 确认后保持追踪，问题还在则重复提醒
- 不记录确认操作

**设计原则**：
- tracker 负责**阈值判断**和**状态管理**（去重、计时）
- notifier 负责**通知执行**（企业微信、本地弹框），不做业务判断
- notifier 是 tracker 的下游，被 tracker 调用

## 5. 状态管理

### 5.1 AttentionEntry 扩展

**文件**：`internal/state/state.go`

```go
type AttentionEntry struct {
    Key          NotificationKey
    FirstSeenAt  time.Time     // 首次发现需要关注的时间
    ConfirmedAt  *time.Time    // 首次确认稳定的时间
    NotifiedAt   *time.Time    // 上次发送通知的时间（企业微信）
    LastAlertedAt *time.Time   // 上次弹框时间（用于叠加提醒）
}
```

**设计说明**：
- `LastAlertedAt` 复用现有 `AttentionEntry` 结构，不新增独立存储
- tracker 在调用 dialog 弹框后更新 `LastAlertedAt`
- 程序重启后状态持久化到磁盘，`LastAlertedAt` 不会丢失

### 5.2 叠加提醒逻辑

```
tracker.ProcessWorkspaces()
      │
      ▼
工作区需要关注 且 达到阈值
      │
      ▼
检查 LastAlertedAt
      │
      ├─── nil（从未弹框）───▶ 弹框
      │
      ├─── 已弹过且未超 RepeatInterval ──▶ 不弹
      │
      └─── 已弹过且已超 RepeatInterval ──▶ 弹框
              │
              ▼
         更新 LastAlertedAt
```

## 6. tracker 逻辑扩展

### 6.1 阈值判断

```go
func (t *Tracker) getThreshold(w api.EnrichedWorkspace) time.Duration {
    if w.Summary.HasPendingApproval {
        return time.Duration(t.approvalThreshold) * time.Minute
    }
    if w.Summary.HasUnseenTurns {
        return time.Duration(t.messageThreshold) * time.Minute
    }
    return 0  // 不需要关注
}
```

### 6.2 叠加提醒判断

```go
func (t *Tracker) shouldShowDialog(entry AttentionEntry, now time.Time) bool {
    if entry.LastAlertedAt == nil {
        return true
    }
    elapsed := now.Sub(*entry.LastAlertedAt)
    return elapsed >= t.repeatInterval
}
```

### 6.3 弹框执行

```go
// 在 tracker 中注入 dialog notifier
type Tracker struct {

}

func (t *Tracker) maybeShowDialog(tw TrackedWorkspace, now time.Time) {
    entry, exists := t.current.Entries[notificationKey(tw.Workspace)]
    if !exists {
        return
    }

    if !t.shouldShowDialog(entry, now) {
        return
    }

    // 构建弹框信息
    title := "Kanban Watcher - 需要关注"
    message := buildDialogMessage(tw)
    url := buildOpenURL(tw.Workspace)

    // 弹框
    clicked, err := t.dialogNotifier.ShowAndOpen(title, message, url)
    if err != nil {
        log.Printf("弹框失败: %v", err)
        return
    }

    if clicked {
        // 用户点击"查看"，已打开浏览器
    }

    // 更新 LastAlertedAt（不可变原则：创建新结构）
    updated := entry
    updated.LastAlertedAt = &now
    t.current = t.current.WithEntry(entry.Key, updated)
}
```

## 7. osascript 弹框实现

### 7.1 DialogNotifier

```go
package notify

import (
    "fmt"
    "os/exec"
    "strings"
)

type DialogNotifier struct {
    // dialog notifier 只负责执行弹框，不管理状态
}

func NewDialogNotifier() *DialogNotifier {
    return &DialogNotifier{}
}

// ShowAndOpen 显示弹框，根据用户选择决定是否打开浏览器
// 返回：用户是否点击了"查看"
func (d *DialogNotifier) ShowAndOpen(title, message, url string) (bool, error) {
    script := fmt.Sprintf(
        `display dialog "%s" with title "%s" buttons {"查看", "忽略"} default button "查看"`,
        escapeAppleScript(message),
        escapeAppleScript(title),
    )

    cmd := exec.Command("osascript", "-e", script)
    output, err := cmd.CombinedOutput()
    result := strings.TrimSpace(string(output))

    // 解析结果判断用户点击的按钮
    if strings.Contains(result, "查看") {
        // 用户点击"查看"，打开浏览器
        if err := exec.Command("open", url).Run(); err != nil {
            return true, fmt.Errorf("打开浏览器失败: %w", err)
        }
        return true, nil
    }

    if strings.Contains(result, "忽略") {
        return false, nil
    }

    // 其他情况（错误、超时等），视为用户取消
    return false, fmt.Errorf("unexpected dialog result: %s", result)
}

// escapeAppleScript 转义 AppleScript 特殊字符
func escapeAppleScript(s string) string {
    s = strings.ReplaceAll(s, `"`, `\"`)
    s = strings.ReplaceAll(s, `\`, `\\`)
    return s
}
```

## 10. 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 系统睡眠/休眠后恢复 | 下次轮询时 `elapsed` 会自动累加，tracker 按正常流程处理 |
| 多个工作区同时需要弹框 | 串行弹框，第一个关闭后才弹第二个 |
| osascript 权限被拒 | 返回 error，打印日志，用户下次轮询会再尝试 |
| 弹框被系统遮挡 | macOS 通知中心会显示，点击可重新呼起 |
| 程序重启 | 状态持久化到磁盘，`LastAlertedAt` 不会丢失 |
