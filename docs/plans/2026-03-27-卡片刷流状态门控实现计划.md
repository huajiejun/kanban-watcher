# 卡片刷流状态门控 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让前端仅在工作区仍需刷流时接收 websocket 增量消息，避免已结束卡片继续刷新并触发无效滚动。

**Architecture:** 在 `workspace-home` 的实时消息追加入口增加卡片级门控函数，基于工作区状态与最后一条归一化消息是否处于终态决定是否继续合并消息。保持现有 websocket 连接和消息分组逻辑不变。

**Tech Stack:** TypeScript, Lit, Vitest, Vite
