# 后端错误输出到控制台 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 `scripts/start-dev.sh` 在后台启动后端时，仍能把后端 stderr 同步打印到当前控制台，便于定位后端异常退出原因。

**Architecture:** 保留现有后台启动与日志落盘方式，只为后端 stderr 增加一条“控制台镜像”通道；通过脚本级回归测试先证明当前实现无法在控制台看到错误，再补最小实现和验证。

**Tech Stack:** Bash, Vitest-free shell test, `tee`, `mkfifo`

---

## Chunk 1: 测试先行

### Task 1: 编写脚本级失败测试

**Files:**
- Create: `scripts/test_start_dev_backend_console_logging.sh`
- Modify: `scripts/start-dev.sh`

- [ ] **Step 1: 写失败测试**

用假 `go` / `npx` / `lsof` 命令运行 `scripts/start-dev.sh start <worktree_id>`，让假后端立即向 stderr 输出一行错误，断言：

```bash
stderr 中包含 "模拟后端报错"
日志文件中也包含 "模拟后端报错"
```

- [ ] **Step 2: 运行测试确认失败**

Run: `bash scripts/test_start_dev_backend_console_logging.sh`
Expected: FAIL，提示当前控制台没有收到后端错误输出

## Chunk 2: 最小实现

### Task 2: 为后端 stderr 增加控制台镜像

**Files:**
- Modify: `scripts/start-dev.sh`

- [ ] **Step 1: 增加后端 stderr 镜像辅助变量**

补充后端日志文件、stderr pipe、镜像进程 PID 文件等路径变量。

- [ ] **Step 2: 启动镜像通道**

在启动后端前创建 FIFO，并启动 `tee`，把后端 stderr 同步写入：

```bash
当前控制台 stderr
/tmp/kanban-backend-<port>.log
```

- [ ] **Step 3: 停止与失败时清理镜像资源**

在 `stop_services` 与启动失败分支中清理旧 tee 进程、旧 pipe，避免遗留资源影响下次启动。
