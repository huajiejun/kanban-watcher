# 完善创建工作区功能实现计划

## 需求概述
扩展 kanban-watcher 的创建工作区功能，支持选择 agent、预设选项、模型等配置，并代理上游 vibe-kanban 的创建工作区接口。

## 上游接口分析

### 主要接口
- **路径**: `POST /api/workspaces/start`
- **功能**: 创建并启动工作区
- **请求体**: `CreateAndStartWorkspaceRequest`

### 请求参数结构
```typescript
interface CreateAndStartWorkspaceRequest {
  name: string | null;              // 工作区名称
  repos: Array<WorkspaceRepoInput>; // 仓库列表
  linked_issue: LinkedIssueInfo | null; // 关联任务信息
  executor_config: ExecutorConfig;  // 执行器配置（核心）
  prompt: string;                   // 工作区提示词
  image_ids: Array<string> | null;  // 图片附件ID列表
}

interface ExecutorConfig {
  executor: BaseCodingAgent;        // Agent类型
  variant?: string | null;          // 预设变体（DEFAULT, PLAN等）
  model_id?: string | null;         // 模型ID
  agent_id?: string | null;         // 代理ID
  reasoning_id?: string | null;     // 推理力度
  permission_policy?: PermissionPolicy | null; // 权限策略
}

enum BaseCodingAgent {
  CLAUDE_CODE = "CLAUDE_CODE",
  AMP = "AMP",
  GEMINI = "GEMINI",
  CODEX = "CODEX",
  OPENCODE = "OPENCODE",
  CURSOR_AGENT = "CURSOR_AGENT",
  QWEN_CODE = "QWEN_CODE",
  COPILOT = "COPILOT",
  DROID = "DROID"
}

enum PermissionPolicy {
  AUTO = "AUTO",
  SUPERVISED = "SUPERVISED",
  PLAN = "PLAN"
}
```

## 实现步骤

### 阶段1：类型定义和API层
1. 在 `types/issue.ts` 添加创建工作区相关类型
   - `CreateWorkspaceRequest`
   - `ExecutorConfig`
   - `BaseCodingAgent` enum
   - `PermissionPolicy` enum
   - `WorkspaceRepoInput`
   - `LinkedIssueInfo`

2. 在 `lib/issue-api.ts` 添加API函数
   - `createAndStartWorkspace()` - 创建并启动工作区
   - `linkWorkspaceToIssue()` - 关联工作区到任务

### 阶段2：扩展对话框UI
在 `kanban-watcher-card.ts` 中扩展 `renderCreateWorkspaceDialog()`：

1. **Agent选择器**
   - 显示选项：Claude Code、AMP、Gemini、Codex 等
   - 使用下拉选择或按钮组
   - 默认选择：CLAUDE_CODE

2. **预设选择器**
   - 选项：DEFAULT、PLAN、ROUTER 等
   - 根据选择的agent动态更新可用预设
   - 默认：DEFAULT

3. **模型选择器**
   - 可选，不选则使用默认值
   - 输入框或下拉选择

4. **权限策略选择器**
   - 选项：AUTO、SUPERVISED、PLAN
   - 默认：AUTO

5. **仓库选择（简化版）**
   - 先使用默认仓库，后续可扩展多仓库选择

6. **需求提示词输入**
   - 文本域输入框
   - 可选，用于给agent的初始提示

### 阶段3：实现创建逻辑
1. 收集表单数据
2. 构建 `CreateWorkspaceRequest` 对象
3. 调用API创建工作区
4. 如有关联任务，调用关联API
5. 刷新工作区列表
6. 关闭对话框并显示成功提示

### 阶段4：样式完善
1. 添加选择器组件样式
2. 表单布局优化（分组、间距）
3. 移动端适配

## 技术细节

### 表单验证
- 工作区名称：必填
- Agent：必填，有默认值
- 其他：可选

### 错误处理
- API调用失败显示错误提示
- 网络错误处理
- 表单验证错误提示

### 用户体验
- 创建按钮显示加载状态
- 创建成功后自动刷新工作区列表
- 支持按ESC键关闭对话框

## 文件修改清单
1. `src/types/issue.ts` - 添加类型定义
2. `src/lib/issue-api.ts` - 添加API函数
3. `src/kanban-watcher-card.ts` - 扩展对话框和逻辑
4. `src/styles.ts` - 添加样式

## 优先级
1. 高：类型定义和API层
2. 高：基础表单字段（agent、preset）
3. 中：高级选项（model、permission）
4. 低：仓库选择（可先使用默认）
