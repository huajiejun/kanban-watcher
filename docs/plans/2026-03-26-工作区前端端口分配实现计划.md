# 工作区前端端口分配实现计划

> **For agentic workers:** REQUIRED: Use `superpowers:subagent-driven-development` (if subagents available) or `superpowers:executing-plans` to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让前端启动脚本通过后端接口获取 `6020-6030` 范围内的可用前端端口，并将端口持久化到 `kw_workspaces.frontend_port`，后端端口统一按 `frontend_port + 10000` 推导。

**Architecture:** 在 `kw_workspaces` 上新增 `frontend_port` 字段，由 store/service 负责端口复用、失效重分配与归档释放。HTTP 层新增一个最小接口供 `scripts/start-dev.sh` 调用，脚本不再本地计算前端端口池，只消费接口结果并推导后端端口。

**Tech Stack:** Go、MySQL、Bash、Vitest、现有 `internal/store` / `internal/server` 结构
