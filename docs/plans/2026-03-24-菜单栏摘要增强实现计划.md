# 菜单栏摘要增强 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 macOS 菜单栏下拉增加“任务名 + 单行摘要”显示，优先展示原因提示，其次展示最近一条用户可读消息。

**Architecture:** 上游数据模型负责产出菜单摘要候选，托盘层只负责单行清洗、截断与两行菜单展示。第一版优先保证“原因提示”稳定可用，同时兼容已有 `last_message`/最近消息字段作为增强来源。

**Tech Stack:** Go, getlantern/systray, Go testing

---

- [ ] **Step 3: 写最小实现**

在 `internal/api/types.go` 增加候选字段，例如：

```go
type Workspace struct {
	// ...
	LastMessage *string `json:"last_message"`
}

type EnrichedWorkspace struct {
	Workspace
	Summary       WorkspaceSummary
	DisplayName   string
	MenuSummary   string
	MenuSummaryBy string
}
```

在 `internal/api/menu_summary.go` 实现：

- `buildReasonHint(summary WorkspaceSummary) string`
- `cleanMenuSummary(text string) string`
- `buildMenuSummary(ws Workspace, summary WorkspaceSummary) (text string, source string)`

要求：

- 不在这里做 32 字符截断，只做“可读摘要候选”生成
- `source` 可取 `reason` / `last_message` / `empty`
