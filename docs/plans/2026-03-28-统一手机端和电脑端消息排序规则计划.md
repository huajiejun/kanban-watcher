# 统一手机端和电脑端消息排序规则计划

## 目标

把手机端 `kanban-watcher-card` 和电脑端 `workspace-home` 的消息排序收口为同一套规则，避免同一批 realtime 消息在两端出现顺序不一致，尤其是“后发消息排到前面”的问题。

## 根因

- 电脑端当前优先按 `processId + entryIndex`、再按 `messageId`、最后按 `timestamp` 排序。
- 手机端当前主要按 `timestamp` 排序，最后退回到 `identity` 排序。
- 两端 realtime 追加消息时都可能对同一批消息做重排，因此分叉的 comparator 会直接表现成顺序不一致。

## 方案

1. 在共享消息工具层提取统一的消息排序 comparator。
2. 明确统一排序优先级：
   - 同 `processId` 时按 `entryIndex`
   - 否则按 `messageId`
   - 否则按 `timestamp`
   - 最后按稳定 identity 兜底
3. 电脑端和手机端都改为调用共享 comparator。
4. 先补失败测试，再实现，再回归两端相关测试。

## 验证

- 新增共享排序规则单测，覆盖：
  - 同 process 不同 entryIndex
  - 不同 process 按 messageId
  - 时间戳兜底
  - identity 兜底
- 回归 `workspace-home` 与 `kanban-watcher-card` 相关测试，确认没有引入行为回退。
