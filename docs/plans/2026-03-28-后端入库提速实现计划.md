# 后端入库提速 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为后端同步链路引入 `200ms` 的 process entry 批量缓冲和配套降频机制，在不改前端逻辑的前提下显著降低数据库写入压力。

**Architecture:** 在 `internal/sync` 中新增进程级写入缓冲器，将 `consumeProcessLogs` 的逐条查库和逐条落库改为按 `process_id` 聚合后批量刷库，并把订阅位点推进放到批量写入成功之后。同时对工作区运行态刷新和快照推送增加节流，减少额外数据库与广播开销。

**Tech Stack:** Go, `database/sql`, MySQL, WebSocket, Go testing, sqlmock
