# 快捷按钮功能设计文档

## 需求概述

在弹窗对话区域的输入框上方增加快捷按钮，实现点击即发送的快速回复功能。

## 功能规格

### 1. 通用快捷词（始终显示）

| 按钮 | 说明 |
|------|------|
| 继续 | 常用回复 |
| 同意 | 常用回复 |

### 2. 动态方案按钮（智能识别）

根据最后一条 AI 消息内容，自动识别并显示方案/选项按钮。

**识别规则（正则匹配）：**

| 格式类型 | 正则模式 | 示例 |
|----------|----------|------|
| 中文数字方案 | `方案[一二三四五六七八九十]+` | 方案一、方案二 |
| 阿拉伯数字方案 | `方案\s*[1-9]` | 方案1、方案 2 |
| 中文数字选项 | `选项[一二三四五六七八九十]+` | 选项一、选项二 |
| 字母选项 | `选项\s*[A-Za-z]` | 选项A、选项 B |
| 英文选项 | `Option\s*[A-Za-z]` | Option A、OptionB |
| 序号列表 | `^\s*[1-9]\.` | 1.、2.、3. |
| 括号字母 | `\([A-Za-z]\)` | (A)、(B)、(C) |

**限制规则：**
- 只检测最后一条 AI 消息
- 去重处理
- 最多显示 5 个动态按钮
- 按出现顺序排列

### 3. UI 设计

**位置：** 输入框上方，消息列表下方

**布局：**
```
┌─────────────────────────────────────┐
│  [消息列表区域]                      │
├─────────────────────────────────────┤
│  [继续] [同意] [方案一] [方案二] ...  │  ← 快捷按钮区
├─────────────────────────────────────┤
│  ┌─────────────────────────────┐    │
│  │ 输入框                       │    │
│  └─────────────────────────────┘    │
│  [发送消息] [加入队列]               │
└─────────────────────────────────────┘
```

**按钮样式：**
- 通用按钮（继续、同意）：实心背景，主色调
- 动态按钮（方案/选项）：描边样式，透明背景

### 4. 交互行为

1. **点击即发送**：无需确认，直接发送按钮文本
2. **发送后**：清空输入框，显示发送反馈
3. **按钮隐藏**：当 workspace 正在运行时，动态按钮隐藏（避免干扰）

## 测试用例

### 动态识别测试

| AI 消息内容 | 期望识别结果 |
|-------------|--------------|
| "请选择方案一或方案二" | ["方案一", "方案二"] |
| "选项A是X，选项B是Y" | ["选项A", "选项B"] |
| "1. 方案A\n2. 方案B\n3. 方案C" | ["1.", "2.", "3."] |
| "你可以选择(A)或(B)" | ["(A)", "(B)"] |
| "这是普通消息没有选项" | [] |

### 边界情况

- 消息为空：只显示通用按钮
- 识别超过5个：只显示前5个
- 重复选项：去重后显示
