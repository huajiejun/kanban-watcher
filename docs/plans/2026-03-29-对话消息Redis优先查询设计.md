# 对话消息 Redis 优先查询设计

**日期**: 2026-03-29
**状态**: 已确认

## 目标

优化工作区对话消息加载性能：从 Redis 直接读取最新消息（尚未 flush 到 MySQL 的数据也可立即可见），默认显示最近 50 条，支持上划 cursor 分页加载更多，跨多个 processID 汇总。

## 当前架构

```
前端点击工作区 → /api/workspaces/{id}/latest-messages
  → MySQL process_entries 表（全量加载）
  → 无分页，一次性返回所有消息
```

痛点：Redis buffer 中的最新数据需等 flush 才可见；无分页机制。

## 方案：Redis 优先 + MySQL 兜底

### Redis 数据结构变更

新增 workspace 级 process 索引：

```
workspace_processes:{workspaceID}  →  ZSET
  member: processID
  score:  最新消息的 entry_timestamp UnixMilli
```

现有 Key 不变：
- `process_entries:{processID}` → ZSET
- `process_entry_data:{processID}` → Hash
- `process_last_idx:{processID}` → String

### 写入侧

在 `RedisBuffer.Enqueue` 的 Pipeline 中追加一步 ZADD：

```go
wsZsetKey := "workspace_processes:" + entry.WorkspaceID
pipe.ZAdd(ctx, wsZsetKey, &redis.Z{
    Score:  float64(entry.EntryTimestamp.UnixMilli()),
    Member: processID,
})
pipe.Expire(ctx, wsZsetKey, rb.opts.TTL)
```

ZADD 对同一 member 幂等，score 更新为最新消息时间。

### 查询 API

新增端点：

```
GET /api/workspaces/{id}/messages?limit=50&before={cursor}
```

参数：
- `limit`: 默认 50，最大 200
- `before`: cursor（最后一条消息的 entry_timestamp RFC3339），不传则取最新

查询流程：

1. `ZREVRANGE workspace_processes:{workspaceID} 0 -1` → 按 score 倒序的 processID 列表
2. 对每个 processID（倒序遍历，最新优先）：
   - `ZREVRANGEBYSCORE process_entries:{processID} +inf {cursor} LIMIT 0 {remaining}`
   - `HMGET process_entry_data:{processID} {members}`
   - 反序列化为 ProcessEntry
3. 合并所有 process 的结果，按 entry_timestamp 倒序，取前 limit 条
4. Redis 数据不足 limit 时，调用 `dbStore.GetSessionMessages` 从 MySQL 补充
5. 用 `(processID, entryIndex)` 去重（Redis 和 MySQL 可能有重叠）
6. 返回：`{ messages: [...], has_more: bool, cursor: "..." }`

### 前端集成

**改动文件**：
- `src/lib/http-api.ts`：新增 `fetchWorkspaceMessages({workspaceId, limit, before})`
- `src/web/workspace-home.ts`：
  - `loadWorkspaceMessages` 改为调用新 API（默认 limit=50）
  - 新增 `loadMoreMessages(workspaceId)` 处理上划
  - 新增 `cursorByWorkspace` 和 `hasMoreByWorkspace` 状态
- `src/components/workspace-conversation-pane.ts`：滚动到底部触发 `loadMore`

**不改的部分**：
- 现有 `/api/workspaces/{id}/latest-messages` API 保持不动
- Realtime WebSocket 推送逻辑不变
- 消息渲染组件（DialogMessage）不变

### 数据流

```
首次加载：
  GET /api/workspaces/{id}/messages?limit=50
  → Redis 读最新 → 不够则 MySQL 补充 → 返回 50 条 + cursor

上划加载更多：
  GET /api/workspaces/{id}/messages?limit=50&before={cursor}
  → Redis + MySQL 查更早消息 → 返回 50 条 + 新 cursor

Realtime 新消息：
  WebSocket 推送 prepend 到列表顶部（不变）
```
