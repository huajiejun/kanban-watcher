# 新增任务弹窗设计

## 背景

移动端看板（Mobile Kanban）目前支持任务的查看和编辑，但缺少新建任务的能力。vibe-kanban 上游接口已提供 `POST /api/issues/` 创建任务接口，需要在前端修复并完善新建任务弹窗。

## 目标

在 mobile-card 模式下，通过看板底部 FAB 按钮打开新建任务弹窗，用户填写信息后提交创建，成功后自动刷新看板。

## 约束

- 不做超越 vibe-kanban 接口的功能（纯代理模式）
- 不修改上游 `kanban-watcher-card` 代码
- 保持移动端交互风格与现有详情面板一致

## 字段定义

根据 `CreateIssuePayload`（上游接口）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 任务标题 |
| description | string | 否 | 任务描述，支持 Markdown |
| status_id | string | 否 | 创建后所属状态 |
| priority | string | 否 | 优先级：urgent / high / medium / low |

## 现有问题

1. **FAB 按钮点击无响应**：`mobile-kanban-board.openCreateDialog` 通过 `dispatchEvent` 向 `<mobile-issue-create-dialog>` 发送 `open` 事件，但当 `selectedProjectId` 为空或加载中时，对话框可能未渲染到 DOM 中
2. **弹窗 UI 风格不统一**：现有 `mobile-issue-create-dialog` 的样式与详情面板的底部滑出弹窗不一致
3. **创建成功后未刷新看板**：`issue-created` 事件发出后，看板未重新加载 issues
4. **缺少错误提示**：创建失败时仅在控制台打印错误，用户无感知

## 架构方案

### 方案 A：改造现有弹窗组件（采用）

保留并改造 `mobile-issue-create-dialog.ts`：
- 修复打开触发逻辑
- 对齐详情面板的底部弹窗 UI 风格
- 增加 Markdown 编辑/预览切换
- 创建成功后发出事件，由 `mobile-kanban-board` 捕获并重载看板
- 增加错误提示（弹窗内轻提示）

**优点**：已有基础结构，改动最小，风险低
**缺点**：事件驱动链略绕，但在可控范围内

### 方案 B：在看板页面直接内嵌创建面板（未采用）

将创建表单直接内联到 `mobile-kanban-board` 模板中。
**优点**：状态管理更直接
**缺点**：board 组件体积变大，与现有设计模式不一致

## 详细设计

### 1. 修复打开逻辑

在 `mobile-kanban-board.ts` 中确保：
- 仅当 `selectedProjectId` 存在且非加载中时渲染 `<mobile-issue-create-dialog>`
- `openCreateDialog` 内先检查 `dialog` 元素存在性再发事件
- 或直接通过方法调用（若组件暴露 `show()` 方法）替代事件驱动

### 2. 弹窗 UI 改造

`mobile-issue-create-dialog.ts` 的样式和交互对齐 `mobile-issue-detail-panel`：
- 底部滑出式 Sheet（`border-radius: 16px 16px 0 0`，max-height: 85vh）
- 标题输入框 + 描述区域（支持 Markdown 编辑/预览切换）
- 状态选择下拉框（从 board columns 传入，默认选中第一列）
- 优先级选择器（四档色块按钮，与详情页一致）
- 底部操作栏：取消 + 创建按钮

### 3. 数据流

```
用户点击 FAB
  → mobile-kanban-board.openCreateDialog()
  → mobile-issue-create-dialog.visible = true
  → 用户填写 → 点击创建
  → createIssue() 调用 POST /api/issues/
  → 成功：close() + dispatchEvent("issue-created")
  → mobile-kanban-board 捕获事件 → refetchIssues() → 刷新看板
  → 失败：在弹窗内显示错误提示（不关闭弹窗）
```

### 4. 错误处理

- 网络错误：`请求失败，请检查网络连接`
- 400/422：`创建失败，请检查必填项是否完整`
- 其他 5xx：`服务异常，请稍后重试`
- 错误提示显示在弹窗底部、按钮上方，红色文字

### 5. 创建成功后刷新

`mobile-kanban-board` 在捕获 `issue-created` 后：
1. 调用现有的 `loadIssues()`（或等效方法）重新加载看板数据
2. 可选：如果新任务的状态已知，自动滚动到对应列（不做强要求）

## 涉及文件

- `src/components/mobile-issue-create-dialog.ts`（主要改动）
- `src/components/mobile-kanban-board.ts`（修复触发 + 刷新）
- `src/lib/issue-api.ts`（可能需要增加错误信息处理，已有 `createIssue`）
- `src/styles.ts`（若需要统一弹窗样式变量）

## 验收标准

- [ ] 在有项目的 mobile-card 模式下，点击底部 `+` 按钮能稳定打开新建弹窗
- [ ] 弹窗内可填写标题、描述（Markdown）、状态、优先级
- [ ] 必填校验：标题为空时创建按钮禁用
- [ ] 创建成功：弹窗关闭，看板自动刷新，新任务出现在对应列
- [ ] 创建失败：弹窗内显示明确错误提示，不关闭弹窗
- [ ] 取消/关闭操作：弹窗关闭，表单内容清空
