## 需求描述

在外时通过 URL 查看 Mac 电脑上的文件夹内容（文件、图片、视频等），需要搭建一个文件服务器。

## 技术方案

### 整体架构

```
[Mac 本地]  File Browser (:8080)  ←  FRP Client  ←  FRP Server  →  [外网访问]
```

### 组件说明

| 组件 | 作用 | 端口 |
|------|------|------|
| File Browser | Web 文件管理器 | 8080 |
| Nginx (可选) | HTTPS 反向代理 | 443 |
| FRP Client | 内网穿透客户端 | - |
| FRP Server | 内网穿透服务端 | 7000, 8080 |

## 方案对比

### 方案 A：File Browser + FRP（推荐，简单）

**优点：**
- 部署简单，配置少
- File Browser 自带 Web UI
- 支持图片、视频预览
- 支持文件上传下载

**缺点：**
- 无 HTTPS（需要额外配置 Nginx）
- 认证简单

### 方案 B：File Browser + Nginx + FRP（完整）

**优点：**
- 支持 HTTPS
- 可绑定域名
- 更安全

**缺点：**
- 配置复杂
- 需要域名和 SSL 证书

## 实施步骤

### 第一阶段：本地 File Browser 部署

1. 安装 File Browser
   ```bash
   brew install filebrowser
   ```

2. 创建配置目录
   ```bash
   mkdir -p ~/.filebrowser
   ```

3. 生成密码哈希
   ```bash
   filebrowser hash <your-password>
   ```

4. 编辑配置文件 `~/.filebrowser/config.json`
   ```json
   {
     "port": 8080,
     "address": "127.0.0.1",
     "database": "/Users/用户名/.filebrowser/filebrowser.db",
     "root": "/Users/用户名",
     "username": "admin",
     "password": "<生成的哈希值>"
   }
   ```

5. 启动服务
   ```bash
   ./scripts/filebrowser_manager.sh start
   ```

6. 本地测试
   访问 http://127.0.0.1:8080

### 第二阶段：FRP 内网穿透（可选）

1. 确保有 FRP 服务器

2. 配置 FRP 客户端 `~/.frpc.ini`
   ```ini
   [common]
   server_addr = <FRP服务器IP>
   server_port = 7000
   token = <你的token>

   [filebrowser]
   type = http
   local_ip = 127.0.0.1
   local_port = 8080
   custom_domains = files.your-domain.com
   ```

3. 启动 FRP
   ```bash
   frpc -c ~/.frpc.ini
   ```

### 第三阶段：Nginx HTTPS（可选，完整方案）

1. 安装 Nginx

2. 配置 SSL 证书（使用 Let's Encrypt 或自签名）

3. 配置反向代理（见 `scripts/nginx_filebrowser.conf`）

4. 重载 Nginx
   ```bash
   sudo nginx -s reload
   ```

## 使用场景

### 场景 1：查看照片

1. 将照片文件夹设置为 File Browser 根目录
2. 通过 URL 访问，直接预览

### 场景 2：查看视频

1. File Browser 支持视频流播放
2. 可直接在线观看

### 场景 3：共享文件

1. 生成下载链接
2. 分享给其他人

## 安全建议

1. **使用强密码**：使用 `filebrowser hash` 生成强哈希
2. **限制端口**：只开放必要端口
3. **定期检查日志**：查看异常访问
4. **HTTPS**：生产环境建议使用 Nginx + SSL
5. **IP 白名单**：可在 Nginx 配置 IP 访问限制

## 多文件夹方案

### 方案 1：多实例

为每个文件夹启动不同的 File Browser 实例，使用不同端口：

```bash
# 照片文件夹
filebrowser -p 8081 -r /Users/huajiejun/Photos

# 视频文件夹
filebrowser -p 8082 -r /Users/huajiejun/Videos

# 文档文件夹
filebrowser -p 8083 -r /Users/huajiejun/Documents
```

### 方案 2：单一实例 + 子目录

使用一个 File Browser 实例，根目录设置为主目录：

```
/
├── Photos/      ← 照片
├── Videos/      ← 视频
├── Documents/   ← 文档
└── Downloads/   ← 下载
```

通过目录导航访问不同文件夹。

### 方案 3：使用别名

在主目录创建符号链接：

```bash
ln -s /Volumes/External/Photos ~/MyPhotos
```
