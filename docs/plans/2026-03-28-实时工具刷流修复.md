# 实时工具刷流修复

## 背景

- 文件编辑类 `tool_use` 在实时推流中存在中间增量丢失
- 近期为抑制历史刷流噪音引入的门控，误伤了真实的工具更新
- 空闲工作区前端门控把普通文本误判为终态，可能继续放大丢流现象

## 目标

- 恢复真实 `tool_use` 增量的实时广播
- 保留历史回放与无变化帧的降噪能力
- 修正空闲工作区的实时接收判定，使其与设计文档一致

## 实施步骤

1. 在 `internal/sync/sync_test.go` 增加 `tool_use running` 广播回归测试
2. 在 `tests/workspace-home.test.ts` 增加空闲卡片普通文本尾消息的收流测试
3. 修改后端广播判定，按消息签名变化决定是否广播
4. 修改前端终态判定，普通文本不再视为终态
5. 运行相关 Go 与前端测试验证

## 风险点

- 放开 `tool_use running` 后可能增加实时广播频率
- 需要依赖现有签名去重避免重复帧噪音
