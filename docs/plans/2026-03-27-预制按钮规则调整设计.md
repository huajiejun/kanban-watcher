# 预制按钮规则调整设计

## 背景

当前快捷按钮链路有两个问题：

1. 非方案类提示词偏向“提示用户现在该做什么”，没有聚焦“AI AGENT 接下来可以继续做什么”。
2. 正则兜底提取只覆盖少量固定格式，尤其在编号列表、项目符号列表、自然语言动作拆分场景下不准。

## 目标

让本地 LLM 基于当前消息上下文，给出 1-3 个“AI AGENT 当前可以继续做且容易做”的下一步动作，并支持通过配置控制禁止推荐的动作类型。

## 约束

- 推荐动作必须是 AI AGENT 能直接执行的动作。
- 推荐动作应优先是低风险、低门槛、无需外部审批的动作。
- 部署、合并代码、提交代码、发版、创建或合并 PR 等动作默认禁止。
- 禁止动作需要支持配置化覆盖。

## 方案

### 1. 新增规则配置

在快捷按钮逻辑中新增规则配置对象，至少包含：

- `forbiddenActions`：禁止推荐的动作关键词或短语

默认值内置一组安全黑名单，调用方可通过卡片配置覆盖或追加。

### 2. 调整 Decision Prompt

将非方案类提示词改成：

- 角色：AI AGENT 下一步动作建议器
- 目标：分析当前上下文，输出 1-3 个 AI AGENT 可以继续执行且容易执行的动作
- 限制：不输出部署、合并代码、提交代码、发版等动作

### 3. 强化动作拆分与归一化

保留现有结构化选项提取（如 `方案1`、`选项A`），同时增强以下场景：

- 编号列表：提取 `1. 补充测试` 的动作内容，而不是 `1.`
- 项目符号列表：提取 `- 检查边界条件`
- 自然语言并列动作：按换行、分号、顿号等安全分隔符拆分短动作

### 4. 增加动作过滤

无论动作来自 LLM JSON 还是正则兜底，都统一经过：

- 清洗：去掉序号、引号、尾部标点
- 校验：长度控制、空值过滤
- 安全过滤：移除命中 `forbiddenActions` 的动作
- 去重：保持按钮展示稳定

- `src/lib/quick-buttons.ts`
- `src/types.ts`
- `src/lib/preview-options.ts`
- `src/playground.ts`
- `src/kanban-watcher-card.ts`

- `tests/playground.test.ts`

## 验证重点

1. Decision prompt 改为“AI AGENT 可继续做的事情”导向
2. 禁止动作可通过配置覆盖
3. 编号列表和项目符号列表能提取出真实动作文本
4. 命中禁止动作的推荐会被过滤
