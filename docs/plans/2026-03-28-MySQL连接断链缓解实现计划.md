# MySQL连接断链缓解 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 降低 MySQL 连接被中间链路或远端重置后导致的 `broken pipe` 刷屏概率，并在异常发生时输出可定位的连接池诊断信息。

**Architecture:** 保持现有 `database/sql` + `go-sql-driver/mysql` 架构不变，只增强数据库配置和 store 初始化逻辑。通过给 DSN 增加显式的拨号/读写超时、让连接池寿命参数可配置，以及在可重试连接错误出现时打印连接池统计，先把问题从“无诊断、易刷屏”收敛到“可缓解、可观测”。

**Tech Stack:** Go, `database/sql`, `github.com/go-sql-driver/mysql`, Go testing, sqlmock

```go
func formatDBStats(stats sql.DBStats) string {
	return fmt.Sprintf(
		"open=%d in_use=%d idle=%d wait_count=%d max_idle_closed=%d max_idle_time_closed=%d max_lifetime_closed=%d",
		stats.OpenConnections,
		stats.InUse,
		stats.Idle,
		stats.WaitCount,
		stats.MaxIdleClosed,
		stats.MaxIdleTimeClosed,
		stats.MaxLifetimeClosed,
	)
}
```
