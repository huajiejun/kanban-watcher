# 前端反向代理接入实现计划

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让工作区前端在本地和外网都默认通过自身入口反向代理访问后端 API 和 WebSocket，不再要求单独暴露后端端口。

**Architecture:** 前端默认 `baseUrl` 改为相对路径模式，浏览器统一请求当前页面下的 `/api` 与 `/api/realtime/ws`。开发态由 Vite 代理到 `frontend_port + 10000`，外网态由一个共享的 Nginx 实例根据 `/{frontend_port}/` 路径前缀分别代理前端页面与后端 API/WS。

**Tech Stack:** TypeScript, Lit, Vitest, Vite proxy, Nginx, Bash
