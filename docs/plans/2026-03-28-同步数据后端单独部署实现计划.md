# 同步数据后端单独部署 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 `kanban-watcher` 支持主后端/业务后端角色分离，只有主后端负责同步、实时推送和通知，业务后端仅提供 HTTP 接口并复用同一数据库。

**Architecture:** 在配置层增加运行角色与主 realtime 地址，启动时统一解析角色能力并据此门控同步、realtime 与通知。前端继续用当前 `baseUrl` 发 HTTP，请求 realtime 时改读 `/api/info` 提供的主后端地址。

**Tech Stack:** Go、Lit、TypeScript、Vitest、Go test

---
