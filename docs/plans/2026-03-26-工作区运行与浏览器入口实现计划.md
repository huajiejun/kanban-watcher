# 工作区运行与浏览器入口 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 `kanban-watcher` 的工作区卡片支持启动预设 dev server，并让工作区主页支持打开当前激活工作区的预设穿透地址。

**Architecture:** 后端扩展活跃工作区摘要，显式返回 `browser_url`，并在现有 `/api/workspace/{id}/...` 动作体系内新增启动 dev server 入口。前端在 `workspace-section-list` 中把卡片拆成主操作与动作区 sibling 结构，在 `workspace-home` 中增加“打开浏览器”按钮与启动状态管理，并复用现有工作区刷新链路完成状态回流。

**Tech Stack:** Go HTTP API, Go store layer, Lit, TypeScript, Vitest

---
