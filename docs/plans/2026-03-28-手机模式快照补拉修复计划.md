# 手机模式快照补拉修复计划

**Goal:** 修复手机模式在弹窗已打开、`latest_session_id` 未变化但 `workspace_snapshot` 持续更新时不刷新消息的问题，并将该补偿刷新策略同步到 PC 端，避免两套链路再次分叉。

**Architecture:** 在共享 `realtime-sync` helper 中补充“选中工作区消息版本变化”判定；`kanban-watcher-card` 和 `workspace-home` 在消费 `workspace_snapshot` 时复用该判定，对当前打开的工作区执行一次补偿消息拉取。

**Tech Stack:** TypeScript、Lit、Vitest
