# 弹出卡片优化实施计划

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在工作区弹窗消息流中插入工具调用摘要卡片，并优先优化手机端的一行摘要与展开详情体验。

**Architecture:** 复用现有弹窗消息列表，在前端消息归一化阶段新增 `tool_use` 分支，生成统一的弹窗消息项模型。渲染层按消息类型分别输出用户消息、AI 消息和工具摘要卡片，样式层通过弱化配色和移动端一行布局控制视觉层级。

**Tech Stack:** Lit、TypeScript、Vitest、Home Assistant Lovelace card 样式

---

新增断言：

- [ ] **Step 2: 运行单测，确认失败**

Run: `npm test -- --run tests/kanban-watcher-card.test.ts`
Expected: 找不到工具卡片类名或文本结构。

- [ ] **Step 3: 重构消息列表渲染分支**

将当前消息渲染：

```ts
messages.map((message) => html`...`)
```

改为按 `message.kind` 分支渲染：

- `kind === "message"`：复用现有气泡
- `kind === "tool"`：渲染摘要卡片按钮

建议拆分函数，避免 `src/kanban-watcher-card.ts` 继续变大：

```ts
private renderDialogEntry(message: DialogMessage)
private renderPlainMessage(message: Extract<DialogMessage, { kind: "message" }>)
private renderToolMessage(message: Extract<DialogMessage, { kind: "tool" }>)
```

- [ ] **Step 4: 添加工具卡片展开状态**

在组件状态中新增按消息 key 记录的展开集合，例如：

```ts
private expandedToolMessageKeys = new Set<string>();
```

点击工具卡片时切换展开状态，展开后渲染详情区域。

- [ ] **Step 5: 渲染工具详情区域**

详情区展示：

- 完整摘要
- 命令或路径
- `content` 中的输出文本

长文本仍需做 `word-break` 与代码块滚动保护。

Run: `npm test -- --run tests/kanban-watcher-card.test.ts`
