# 发送消息中介实现计划 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 Home Assistant 只向 `kanban-watcher` 发送简单消息，由 `watcher` 从数据库补齐 `session_id` 与 `executor_config`，并统一支持立即发送和加入队列。

**Architecture:** 后端新增消息上下文持久化与统一分发服务，HTTP 层收敛到 `/api/workspace/{id}/message`，再按 `mode` 转发到 `vibe-kanban` 的 `follow-up` 或 `queue`。前端卡片移除本地伪队列，改为调用 watcher 的真实消息接口，并展示来自后端的发送结果。

**Tech Stack:** Go, MySQL, TypeScript, Lit, fetch API, Go testing, npm test
